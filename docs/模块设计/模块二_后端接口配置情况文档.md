## 二、核心接口详情

### 2.1 会话管理（5个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/session` | POST | 创建会话 |
| `/api/v1/session` | GET | 获取会话列表 |
| `/api/v1/session/{session_id}` | GET | 获取会话详情 |
| `/api/v1/session/{session_id}` | PUT | 更新会话 |
| `/api/v1/session/{session_id}` | DELETE | 删除会话 |

### 2.2 消息管理（3个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/chat/message` | POST | 发送消息 |
| `/api/v1/chat/stream` | POST | 流式发送消息（SSE） |
| `/api/v1/session/{session_id}/messages` | GET | 获取会话消息 |

### 2.3 交互功能（3个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/chat/message/{message_id}/feedback` | POST | 提交反馈 |
| `/api/v1/chat/message/{message_id}/favorite` | POST | 收藏消息 |
| `/api/v1/chat/recommendations` | GET | 获取推荐问题 |

### 2.4 收藏管理（4个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/favorites` | GET | 获取收藏列表 |
| `/api/v1/favorites/{message_id}` | POST | 收藏消息 |
| `/api/v1/favorites/{message_id}` | DELETE | 取消收藏 |
| `/api/v1/favorites/{message_id}/check` | GET | 检查收藏状态 |

### 2.5 知识图谱（14个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/knowledge/entity` | POST | 创建实体 |
| `/api/v1/knowledge/entity/{entity_id}` | GET | 获取实体详情 |
| `/api/v1/knowledge/entity/{entity_id}` | PUT | 更新实体 |
| `/api/v1/knowledge/entity/{entity_id}` | DELETE | 删除实体 |
| `/api/v1/knowledge/relationship` | POST | 创建关系 |
| `/api/v1/knowledge/search` | POST | 搜索实体 |
| `/api/v1/knowledge/entity/{entity_id}/relations` | GET | 获取实体关系 |
| `/api/v1/knowledge/graph` | GET | 获取图谱数据 |
| `/api/v1/knowledge/path` | POST | 查找路径 |
| `/api/v1/knowledge/stats` | GET | 获取统计信息 |

## 三、数据模型

### 3.1 核心模型

**Session（会话）**
- id, user_id, title, created_at, updated_at, message_count, is_pinned

**Message（消息）**
- id, session_id, role, content, created_at, feedback, is_favorite

**Entity（实体）**
- id, name, type, description, region, period, coordinates, meta_data, importance, created_at, updated_at

**Relationship（关系）**
- id, source_id, target_id, relation_type, weight, meta_data, created_at

### 3.2 辅助模型

- MessageSource（消息来源）
- MessageEntity（消息实体）
- MessageKeyword（消息关键词）
- FavoriteQuestion（收藏问题）
- SearchHistory（搜索历史）
- Favorite（收藏）

## 七、部署说明

### 7.1 开发环境

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

cd frontend
npm install
npm run dev
```

### 7.2 生产环境

- 后端：配置PostgreSQL、启用HTTPS、使用Gunicorn部署
- 前端：构建生产版本、部署到静态服务器

### 7.3 访问地址

- 后端API：http://localhost:8000
- 前端界面：http://localhost:5173
- API文档：http://localhost:8000/docs

## 八、已知限制

1. **用户认证缺失**：使用默认用户ID，不支持多用户
2. **AI响应依赖**：需要配置DASHSCOPE_API_KEY
3. **NLP功能待完善**：实体识别、关键词提取使用mock数据
4. **向量检索缺失**：未实现基于向量相似度的检索
