## 二、核心接口详情

### 2.1 会话管理（6 个）

| 接口                                    | 方法   | 功能         | 说明                       |
| --------------------------------------- | ------ | ------------ | -------------------------- |
| `/api/v1/session`                       | POST   | 创建会话     | 返回完整会话对象           |
| `/api/v1/session`                       | GET    | 获取会话列表 | 支持分页 (page, page_size) |
| `/api/v1/session/{session_id}`          | GET    | 获取会话详情 | 返回单个会话               |
| `/api/v1/session/{session_id}`          | PUT    | 更新会话     | 更新标题等信息             |
| `/api/v1/session/{session_id}`          | DELETE | 删除会话     | 级联删除消息               |
| `/api/v1/session/{session_id}/messages` | GET    | 获取会话消息 | 分页获取消息列表           |

### 2.2 消息管理（3 个）

| 接口                                    | 方法 | 功能                | 说明                                                  |
| --------------------------------------- | ---- | ------------------- | ----------------------------------------------------- |
| `/api/v1/chat/message`                  | POST | 发送消息            | 返回 entities、keywords、relations                    |
| `/api/v1/chat/stream`                   | POST | 流式发送消息（SSE） | 增量推送 content_chunk、entities、keywords、relations |
| `/api/v1/session/{session_id}/messages` | GET  | 获取会话消息        | 分页获取历史消息                                      |

### 2.3 交互功能（3 个）

| 接口                                         | 方法 | 功能          | 说明                      |
| -------------------------------------------- | ---- | ------------- | ------------------------- |
| `/api/v1/chat/message/{message_id}/feedback` | POST | 提交反馈      | feedback: helpful/unclear |
| `/api/v1/chat/message/{message_id}/favorite` | POST | 收藏/取消收藏 | 切换收藏状态              |
| `/api/v1/chat/recommendations`               | GET  | 获取推荐问题  | 返回 4 个推荐问题         |

### 2.4 收藏管理（4 个）

| 接口                                   | 方法   | 功能         | 说明             |
| -------------------------------------- | ------ | ------------ | ---------------- |
| `/api/v1/favorites`                    | GET    | 获取收藏列表 | 分页获取用户收藏 |
| `/api/v1/favorites/{message_id}`       | POST   | 收藏消息     | 添加收藏         |
| `/api/v1/favorites/{message_id}`       | DELETE | 取消收藏     | 删除收藏         |
| `/api/v1/favorites/{message_id}/check` | GET    | 检查收藏状态 | 返回 is_favorite |

### 2.5 图谱快照（6 个）

| 接口                                         | 方法   | 功能             | 说明                              |
| -------------------------------------------- | ------ | ---------------- | --------------------------------- |
| `/api/v1/graph/snapshot`                     | POST   | 创建快照         | 保存图谱数据、entities、relations |
| `/api/v1/graph/snapshot/{snapshot_id}`       | GET    | 获取快照详情     | 返回完整快照数据                  |
| `/api/v1/graph/snapshots`                    | GET    | 获取快照列表     | 分页获取，支持 session_id 过滤    |
| `/api/v1/graph/snapshot/{snapshot_id}`       | DELETE | 删除快照         | 删除指定快照                      |
| `/api/v1/graph/snapshot/{snapshot_id}/share` | POST   | 分享快照         | 生成分享链接                      |
| `/api/v1/graph/session/{session_id}/data`    | GET    | 获取会话图谱数据 | 聚合会话中所有 entities、keywords |

## 三、数据模型

### 3.1 核心模型

**Session（会话）**

- id, user_id, title, created_at, updated_at, message_count, is_pinned

**Message（消息）**

- id, session_id, role, content, created_at, feedback, is_favorite, entities, keywords, relations, sources

**Entity（实体）**

- id, name, type, description, relevance
- type: inheritor | technique | work | pattern | region | period | material

**Relation（关系）**

- id, source, target, type, confidence, evidence, bidirectional
- type: inherits | origin | creates | flourished_in | located_in | uses_material | has_pattern | related_to | influenced_by | contains

**Snapshot（快照）**

- id, session_id, message_id, graph_data, keywords, entities, relations, title, description, created_at, is_shared, share_url

### 3.2 辅助模型

- Source（消息来源）：id, title, content, url, page, relevance
- Favorite（收藏）：id, user_id, message_id, created_at
- Feedback（反馈）：feedback_type, content, rating

## 六、AI 服务系统

### 6.1 系统特点

- **流式响应**：SSE 实时推送，支持增量更新
- **实体识别**：7 种实体类型自动识别
- **关系推断**：10 种关系类型智能推断
- **关键词提取**：自动提取 5-10 个关键词
- **Mock 降级**：API Key 缺失时自动降级到 Mock 响应

### 6.2 实体类型（7 种）

| 类型      | 名称   | 颜色    | 说明          |
| --------- | ------ | ------- | ------------- |
| inheritor | 传承人 | #FF6B6B | 技艺传承者    |
| technique | 技艺   | #4ECDC4 | 传统工艺技术  |
| work      | 作品   | #45B7D1 | 代表作品      |
| pattern   | 纹样   | #F7A072 | 装饰纹样      |
| region    | 地域   | #A8E6CF | 发源/流行地区 |
| period    | 时期   | #DCEDC1 | 历史时期      |
| material  | 材料   | #98D8C8 | 使用材料      |

### 6.3 关系类型（10 种）

| 类型          | 名称     | 说明                  |
| ------------- | -------- | --------------------- |
| inherits      | 传承     | 传承人之间的传承关系  |
| origin        | 起源     | 技艺/作品的起源       |
| creates       | 创作     | 传承人创作作品        |
| flourished_in | 盛行于   | 技艺/作品盛行于某时期 |
| located_in    | 位于     | 作品/技艺位于某地     |
| uses_material | 使用材料 | 作品/技艺使用材料     |
| has_pattern   | 包含纹样 | 作品包含纹样          |
| related_to    | 相关     | 一般相关关系          |
| influenced_by | 受影响的 | 受其他技艺影响        |
| contains      | 包含     | 整体包含部分          |

### 6.4 知识库主题

## 八、已知限制

1. **用户认证缺失**：使用 mock 认证，不支持多用户隔离
2. **AI 响应依赖**：需要配置 DASHSCOPE_API_KEY 才能获得真实 AI 响应
3. **内存存储**：图谱快照使用内存存储，重启后数据丢失
4. **NLP 能力**：实体识别、关系推断基于 LLM，准确率依赖模型能力
5. **图谱规模**：单次请求限制 50 个节点，避免浏览器性能问题
