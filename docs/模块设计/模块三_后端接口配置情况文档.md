## 二、核心接口详情

### 2.1 实体管理（4个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/knowledge/entity` | POST | 创建实体 |
| `/api/v1/knowledge/entity/{entity_id}` | GET | 获取实体详情 |
| `/api/v1/knowledge/entity/{entity_id}` | PUT | 更新实体 |
| `/api/v1/knowledge/entity/{entity_id}` | DELETE | 删除实体 |

### 2.2 关系管理（4个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/knowledge/relationship` | POST | 创建关系 |
| `/api/v1/knowledge/relationship/{relationship_id}` | GET | 获取关系详情 |
| `/api/v1/knowledge/relationship/{relationship_id}` | PUT | 更新关系 |
| `/api/v1/knowledge/relationship/{relationship_id}` | DELETE | 删除关系 |

### 2.3 搜索和检索（6个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/knowledge/search` | POST | 搜索实体 |
| `/api/v1/knowledge/search/advanced` | POST | 高级搜索 |
| `/api/v1/knowledge/graph` | GET | 获取图谱数据 |
| `/api/v1/knowledge/entity/{entity_id}/detail` | GET | 获取实体详情及关联 |
| `/api/v1/knowledge/entity/{entity_id}/relations` | GET | 获取实体关系 |
| `/api/v1/knowledge/path` | POST | 查找实体间路径 |

### 2.4 统计和分析（7个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/knowledge/stats` | GET | 获取统计信息 |
| `/api/v1/knowledge/categories` | GET | 获取实体类型 |
| `/api/v1/knowledge/regions` | GET | 获取地域列表 |
| `/api/v1/knowledge/periods` | GET | 获取时期列表 |
| `/api/v1/knowledge/analysis/connectivity` | GET | 分析图谱连通性 |
| `/api/v1/knowledge/analysis/centrality/{entity_id}` | GET | 分析实体中心性 |
| `/api/v1/knowledge/analysis/community` | 分析社区结构 |

### 2.5 用户交互（7个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/knowledge/favorite` | POST | 添加收藏 |
| `/api/v1/knowledge/favorite` | DELETE | 取消收藏 |
| `/api/v1/knowledge/favorite/{user_id}` | GET | 获取用户收藏列表 |
| `/api/v1/knowledge/favorite/check` | GET | 检查收藏状态 |
| `/api/v1/knowledge/feedback` | POST | 提交反馈 |
| `/api/v1/knowledge/feedback/entity/{entity_id}` | GET | 获取实体反馈 |
| `/api/v1/knowledge/feedback/user/{user_id}` | GET | 获取用户反馈 |

### 2.6 数据导入导出（2个）

## 三、数据模型

### 3.1 核心模型

**Entity（实体）**
- id, name, type, description, region, period, coordinates, meta_data, importance, created_at, updated_at

**Relationship（关系）**
- id, source_id, target_id, relation_type, weight, meta_data, created_at

**Favorite（收藏）**
- id, user_id, entity_id, created_at

**Feedback（反馈）**
- id, user_id, entity_id, feedback_type, content, rating, created_at

### 3.2 辅助模型

- SearchHistory（搜索历史）

## 四、环境配置

### 4.1 当前配置

```env
APP_NAME=非遗数字生命互动引擎
APP_VERSION=1.0.0
DEBUG=True
DATABASE_URL=sqlite+aiosqlite:///./heritage.db
CORS_ORIGINS=["http://localhost:3000","http://localhost:5173"]
JWT_SECRET_KEY=your-secret-key-change-in-production
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

| 配置项 | 用途 | 状态 |
|-------|------|------|
| DATABASE_URL | PostgreSQL数据库 | 可选 |
| REDIS_URL | 缓存 | 可选 |

## 七、部署说明

### 7.1 开发环境

```bash
cd backend
pip install -r requirements.txt
python init_data.py
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

cd frontend
npm install
npm run dev
```

### 7.2 生产环境

- 后端：配置PostgreSQL、启用HTTPS、使用Gunicorn部署
- 前端：构建生产版本、部署到静态服务器

### 7.3 访问地址

- 后端API：http://localhost:8000
- 前端界面：http://localhost:5173
- API文档：http://localhost:8000/docs

## 八、已知限制

1. **数据规模限制**：当前使用SQLite，适合中小规模数据
2. **搜索功能限制**：仅支持关键词搜索，不支持语义搜索
3. **图谱渲染限制**：大规模图谱（>1000节点）可能存在性能问题
