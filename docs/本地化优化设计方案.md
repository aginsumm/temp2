## 一、概述

### 1.1 设计目标

本方案旨在实现非遗数字生命互动引擎的本地化优化，确保：

1. **功能完整性**：除大模型服务外，所有功能均可独立本地运行
2. **数据隐私**：用户数据优先本地存储，确保隐私安全
3. **离线可用**：核心功能在无网络环境下可正常使用
4. **平滑扩展**：预留后端接口，支持未来无缝集成

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端应用层                            │
├─────────────────────────────────────────────────────────────┤
│                      服务路由器 (ServiceRouter)              │
├───────────────────┬───────────────────┬─────────────────────┤
│   本地服务层      │    Mock 服务层    │    远程服务层       │
│  (Local-first)   │   (Fallback)      │  (Backend-required) │
├───────────────────┼───────────────────┼─────────────────────┤
│ chatRepository    │ mockChatService   │  /api/v1/chat/*     │
│ knowledgeRepo     │ mockKnowledgeSvc  │  /api/v1/knowledge  │
│ localAuthService  │ mockLLMService    │  /api/v1/llm/*      │
├───────────────────┴───────────────────┴─────────────────────┤
│                    本地数据存储层                            │
│                   (IndexedDB + localStorage)                 │
└─────────────────────────────────────────────────────────────┘
```

## 三、本地数据存储架构

### 3.1 存储技术选型

| 存储类型 | 技术 | 用途 |
|----------|------|------|
| 结构化数据 | IndexedDB | 会话、消息、实体、关系 |
| 配置数据 | localStorage | 用户设置、Token |
| 缓存数据 | Memory Cache | 临时数据、计算结果 |

### 3.2 IndexedDB 数据模型

```
HeritageAppDB (v3)
├── sessions          # 会话存储
│   ├── id (PK)
│   ├── user_id (Index)
│   ├── updated_at (Index)
│   └── is_pinned (Index)
│
├── messages          # 消息存储
│   ├── id (PK)
│   ├── session_id (Index)
│   ├── created_at (Index)
│   ├── role (Index)
│   └── is_favorite (Index)
│
├── users             # 用户数据
│   ├── id (PK)
│   ├── username (Unique)
│   └── email (Unique)
│
├── entities          # 知识实体 (v3新增)
│   ├── id (PK)
│   ├── type (Index)
│   ├── region (Index)
│   ├── period (Index)
│   ├── importance (Index)
│   └── name (Index)
│
├── relationships     # 知识关系 (v3新增)
│   ├── id (PK)
│   ├── source_id (Index)
│   ├── target_id (Index)
│   └── relation_type (Index)
│
├── searchHistory     # 搜索历史 (v3新增)
│   ├── id (PK)
│   ├── user_id (Index)
│   ├── keyword (Index)
│   └── created_at (Index)
│
├── favorites         # 收藏记录
│   ├── id (PK)
│   ├── user_id (Index)
│   ├── entity_id (Index)
│   └── created_at (Index)
│
├── feedbacks         # 反馈记录
│   ├── id (PK)
│   ├── user_id (Index)
│   ├── entity_id (Index)
│   └── created_at (Index)
│
├── pendingOperations # 待同步操作
│   ├── id (PK)
│   ├── type (Index)
│   └── timestamp (Index)
│
└── settings          # 应用设置
    └── key (PK)
```

### 3.3 数据同步策略

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  本地操作   │────▶│  操作队列   │────▶│  后端同步   │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       ▼                   ▼                   ▼
  IndexedDB          pendingOperations    API Call
  立即存储            异步队列             网络恢复后
```

## 四、核心模块实现

### 4.1 功能配置模块

**文件**: `src/config/featureConfig.ts`

```typescript
// 功能模式定义
export type FeatureMode = 'backend-required' | 'local-first' | 'local-only';

// 功能配置接口
export interface FeatureConfig {
  mode: FeatureMode;
  description: string;
  backendEndpoint?: string;
  localImplementation: string;
  fallbackStrategy?: 'queue' | 'cache' | 'mock' | 'none';
  priority: 'critical' | 'high' | 'medium' | 'low';
  dependencies?: string[];
}
```

### 4.2 服务路由器

**文件**: `src/data/serviceRouter.ts`

核心功能：
- 自动检测后端服务可用性
- 根据配置选择本地或远程服务
- 提供服务降级和 Mock 支持
- 统一的服务状态监控

### 4.3 本地数据库

**文件**: `src/data/localDatabase.ts`

核心功能：
- IndexedDB 封装
- 版本迁移支持
- 批量操作优化
- 事务支持

### 4.4 Mock LLM 服务

**文件**: `src/data/mockServices/mockLLMService.ts`

核心功能：
- 本地大模型模拟
- 非遗知识库集成
- 流式响应支持
- 意图识别和实体提取

## 五、后端扩展指南

### 5.1 扩展步骤

1. **配置环境变量**
   ```bash
   # .env.local
   VITE_API_BASE_URL=https://your-api.com/api/v1
   VITE_LLM_PROVIDER=dashscope
   VITE_DASHSCOPE_API_KEY=your_key
   ```

2. **实现后端接口**
   - 参考 `backend/app/api/v1/` 目录结构
   - 实现对应的 API 端点

3. **更新功能配置**
   ```typescript
   // src/config/featureConfig.ts
   llmChat: {
     mode: 'backend-required',
     // ... 其他配置
   }
   ```

4. **测试验证**

## 八、维护指南

### 8.1 数据库迁移

当需要更新数据库结构时：

1. 更新 `DB_VERSION` 常量
2. 在 `localDatabase.ts` 中添加新的 `createVxStores` 方法
3. 在 `onupgradeneeded` 中调用新方法

### 8.2 添加新功能

1. 在 `featureConfig.ts` 中定义功能配置
2. 实现本地 Repository 或 Service
3. 在 `serviceRouter.ts` 中添加路由逻辑
4. 更新相关 Store 和组件

### 8.3 集成新后端服务

1. 在 `.env.example` 中添加配置项
2. 实现后端 API 接口
3. 更新 `featureConfig.ts` 中的模式配置
4. 在 `serviceRouter.ts` 中实现远程调用逻辑
