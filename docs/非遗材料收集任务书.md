# 非遗材料收集任务书

## 1. 收集目标

本任务书仅针对非遗材料收集，不单独展开地方志全文采集。  
本阶段收集范围统一为 `4个地方 x 4种非遗 = 16种非遗`。

- 地区范围：湖北武汉、陕西西安、江苏苏州、四川成都
- 每种非遗交付标准：`约1500字文字材料 + 8张相关图片`
- 每种非遗最少来源数：`3条`
- 其中来源建议为：`2条非遗官方/权威来源 + 1条地方志或地方文化背景来源`

总量要求：

- 非遗文字总量：约 `24000字`
- 非遗图片总量：`128张`
- 来源链接总量：至少 `48条`

说明：

- 地方志来源用于补充地域背景、历史脉络和事实校对，不单独计入本次“地方志正文收集任务量”。
- 如个别非遗条目资料明显不足，可在同一城市内替换为其他代表性非遗项目，但总数量和交付标准不变。

## 2. 单条非遗收集标准

每种非遗统一按以下标准收集，便于后续数据库设计、知识图谱建模和问答入库：

### 2.1 文字内容

每种非遗提供 `约1500字` 的可用整理稿，建议拆分为：

- 基本简介：`150-200字`
- 历史渊源：`250-300字`
- 核心技艺 / 表演形式 / 工艺特点：`400-500字`
- 代表作品 / 代表场景 / 代表器物：`200-250字`
- 传承人 / 保护现状 / 当代传播：`200-250字`
- 地域文化关联说明：`150-200字`

要求：

- 文字为整理后的可用内容，不直接堆砌原文截图或大段摘抄。
- 内容需尽量覆盖“是什么、从哪来、有什么特点、如何传承”四个核心问题。
- 每种非遗至少整理 `5-8个关键词`，便于后续搜索和标签化。

### 2.2 图片内容

每种非遗提供 `8张` 相关图片，建议结构如下：

- 非遗项目整体展示图：`2张`
- 工艺流程 / 表演过程图：`2张`
- 代表作品 / 细节特写图：`2张`
- 传承人 / 场馆 / 相关场景图：`2张`

要求：

- 图片尽量来自权威公开来源，分辨率尽量清晰。
- 每张图片需记录来源链接和一句简短说明，便于后续展示与归档。
- 同一非遗不要重复提交高度相似图片。

### 2.3 来源要求

每种非遗至少提交 `3条来源`：

- 主来源1：非遗项目文字介绍来源
- 主来源2：非遗项目补充来源或多媒体来源
- 校对来源：地方志 / 方志网 / 地方文化背景来源

## 3. 来源使用说明

本任务书的资料源头说明结合 [原始资料来源清单](/Users/Ray_/Documents/Developer/project24798-6754/docs/原始资料来源清单.md) 执行，统一使用以下来源类别：

### 3.1 非遗主来源

1. 中国非物质文化遗产网  
   链接：<https://www.ihchina.cn/>  
   用途：作为非遗项目名称、类别、简介、代表性内容的首选文字来源。

2. UNESCO非物质文化遗产名录  
   链接：<https://ich.unesco.org/en/lists>  
   用途：用于国际级非遗项目的背景补充、英文名称校对和传播说明。若对应项目不在该名录中，可不强制使用。

3. 国家图书馆“国家级非遗代表性传承人记录工作成果服务”  
   链接：<https://ich.nlc.cn/>  
   用途：优先补充图片、视频、传承人、技艺过程等多媒体材料。

### 3.2 地方背景来源

1. 各省“省情网 / 方志网 / 地方志办公室官网”
2. 国家数字方志馆  
   链接：<http://gjszfzg.cn/#/home>

用途：

- 用于核对地域归属、历史背景、地方文化语境
- 用于补充非遗与城市、地域传统之间的关联说明
- 不要求为每种非遗额外整理长篇地方志正文，只需作为校对和背景依据保留链接

## 5. 交付格式

为便于后续数据库设计、知识图谱建模、RAG入库和前端展示，建议每种非遗按以下字段交付：

| 字段名 | 说明 |
| --- | --- |
| 地区 | 所属城市，如“武汉” |
| 非遗名称 | 项目名称 |
| 非遗类别 | 传统技艺 / 戏曲 / 民间文学 / 传统美术等 |
| 内容正文 | 约1500字整理稿 |
| 关键词 | 5-8个 |
| 图片文件名 | 共8张，按顺序命名 |
| 图片说明 | 每张图片1句说明 |
| 主来源1 | 文字主来源链接 |
| 主来源2 | 补充来源或多媒体来源链接 |
| 校对来源 | 方志网 / 地方志 / 地方文化来源链接 |
| 备注 | 资料缺口、版权说明、待核实项 |

建议命名方式：

- 文字稿：`城市-非遗名称.md`
- 图片文件夹：`城市-非遗名称/`
- 图片命名：`01.jpg` 至 `08.jpg`

## 6. 交接要求

- 所有条目按统一字段整理，避免后续重复清洗。
- 能明确来源的内容优先入库，来源不清的内容单独标记。
- 图片与文字一一对应，确保开发人员能直接建立详情页、图谱节点和检索索引。
- 若后续数据库需要拆表，可直接按“地区、项目、图片、来源、关键词”五类字段拆分。

本任务书执行完成后，应形成一套可直接交给后续开发人员使用的基础非遗数据集。
